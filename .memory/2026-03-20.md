## 关键技术细节

### DuckDB Aggregate State 访问
- `duckdb_aggregate_state` 直接 cast 为 state 指针，不通过 `internal_ptr`
- DuckDB 内部用 `reinterpret_cast<State*>(state_p)`

### Function Set 注册
- 每个 overload 函数必须调用 `duckdb_aggregate_function_set_name()`
- 否则 `duckdb_register_aggregate_function_set()` 静默失败

### C_STRUCT ABI
- 入口：`{name}_init_c_api(info, access) -> bool`
- 通过 `access->get_api(info, "v1.2.0")` 获取 `duckdb_ext_api_v1` 函数指针表
- DuckDB 用 `RTLD_LOCAL` 加载扩展，无法直接解析 DuckDB 符号
- 所有 DuckDB C API 调用必须通过函数指针表
- 元数据 ABI_TYPE 设为 "C_STRUCT"，VERSION_FIELD 设为 API 版本 "v1.2.0"（非 DuckDB 发行版本）

### link-duckdb feature
- 默认启用：静态链接 DuckDB（用于 cargo test）
- 关闭时：通过 ext_api_v1 函数指针表调用（loadable extension）
