## 项目目标
用 Rust + CUDA 实现 GPU 加速 DuckDB 扩展（sirius-rust），参考 [sirius](https://github.com/sirius-db/sirius)。

## 关键决策
- GPU 是核心，不是可选项
- DuckDB v1.5.0 以 git submodule 引入，bindgen 生成 FFI，不用 crates.io
- GPU 通过 `--features gpu` feature gate 可选编译，无 CUDA 环境也能 build/test
- 多规模 TPC-H benchmark（SF0.1 / SF0.5 / SF1 / SF5 / SF10），不只跑 SF1
- SF10 超 4GB VRAM 需分批传输，本身是论文亮点

## sirius 原版架构
- 核心流程：SQL → DuckDB → Substrait Plan → Sirius → GPU 执行 (fallback CPU) → 返回
- 三个入口 Table Function：gpu_buffer_init / gpu_processing / gpu_processing_substrait
- 聚合函数：COUNT, COUNT_STAR, COUNT_DISTINCT, SUM, SUM_NO_OVERFLOW, AVG, MAX, MIN
- 基于 cuDF + RMM，21 个 GPU 加速算子

## sirius C++ 架构（非 CUDA 部分）
1. **查询计划转换**（~17个文件）：Substrait → GPU 物理计划，60+ 种算子
2. **执行管道编排**：gpu_executor / gpu_pipeline / gpu_meta_pipeline
3. **物理算子实现**（~21个）：Filter、Projection、HashJoin、HashAggregate、OrderBy 等
4. **GPU 内存管理**：Buffer Manager，GPU/CPU 混合分配，RMM 内存池
5. **数据表示与类型转换**：DataWrapper、GPUColumn
6. **Fallback 机制**：执行前校验，不支持则回退 CPU
